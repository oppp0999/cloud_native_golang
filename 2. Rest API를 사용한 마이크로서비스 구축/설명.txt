REST API 는 다양한 형태에서 개발된 애플리케이션들이 서로 통신할 수 있게 해준다.

HTTP는 클라이언트-서버, 요청-응답 프로토콜이다. 이는 데이터 흐름이 다음과 같이 동작함을 의미한다.
-HTTP 클라이언트가 HTTP 서버에게 요청을 보낸다.
-HTTP 서버는 들어오는 요청들을 대기해 받는다. 이후 다시 응답
--HTTP 클라이언트 요청
: 클라이언트는 서버에게 사원을 요청
: 클라이언트는 서버상의 자원 추가/수정을 요청

--------------------------------------------------------------------------------
--------------------------------------------------------------------------------
요청 방식 -> 요청, 수정, 추가, 삭제까지 범위
공통적인 메소드

GET -> 웹 애플리케이션에 널리 사용되는 HTTP 메서드로 자원 요청하는 방법에 대한 것

POST -> 자원 업데이터 신규 생성, 요청 바디 부분에 이 새로운 이벤트 정보 전부를 추가하는 것

PUT -> 자원 생성하거나 덮어쓰기 위한 메서드, POST와 다르게 전에 존재하지 않은 자원을 요청으로 보내 새로운 자원을 생성

DELETE -> 자원을 삭제하고자 사용
